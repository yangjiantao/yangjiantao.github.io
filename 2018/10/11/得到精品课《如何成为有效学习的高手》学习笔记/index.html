得到精品课《如何成为有效学习的高手》学习笔记

发表于 2018-10-11 | 分类于 随笔 | 阅读次数:

高效学习定义

高效学习的定义在我看来是找到最适合自己的学习手法。在相对短的时间内集中注意力，以解决一个工作和生活当中的难题为目的着手学习。你需要勤加练习，有时候还需要向名师求助。

  • 几个关键点

    一，适合自己；二，学习手法；三，短时间；四，注意力；五，解决难题；六，设定目标；七，名师。

  • Summary

    在短时间内（比如两周）集中注意力，设定一个预估可执行的任务或难题，用适合自己的方法（获取知识途径）完成学习目标。

找到适合自己的学习方法

两种学习方法：自然主义和结构主义。

  • 自然主义是模仿式的学习，遇到什么就学什么，积少成多，以量取胜。学完了就那样了，它就是你自己的了，你就可以去运用了。
  • 结构主义是说学东西的时候更注重一个结构，体系化的去学习。学会之后的目的，你要注意，一定是为了创造，是一种质变，它要的是一种质变。

更好的学习方法

怎么样做是更好的学习方法？最简单的就是把你的目标要拆碎，针对那些细节进行反复的练习。不要一开始就照顾全局。

碎片化学习的真相

  • 碎片化学习的真相是碎片化学习一定要基于搜索，搜索是什么意思？意思就是你要带着困惑和问题去执行碎片化的学习。

  • 带着困惑和问题去执行碎片化的学习，最终的效果一定是要把这些碎片给链接起来。在一段时间内，你的头脑当中应该始终有一个课题，这些碎片它不是满地的落叶，而是我们可以把它比喻成一个打碎的瓷器所产生的碎片。你要有目的的去拾获这些碎片，而不是乱捡。就算碎片化知识是树叶，你也得先有树根、树干和树枝。 碎片化学习要的是枝繁叶茂，而不是捡几片树叶回来夹在书里边。

Summary

  • 复杂（不明确）任务、大目标，首先是将其拆分成可执行的小任务/目标。
  • 短时间内集中注意力将小目标各个击破。
  • 带着困惑和问题去执行碎片化学习，且最终要将这些碎片给链接起来。
  • 先有自己的技能体系地图（树根、树干），再用碎片化学习来补充（树枝、树叶）。
  • 收藏是一种幻觉，收再多也跟你没关系，屯书也一样。

任务驱动学习

  • 成年人要放弃一种需求，就是学什么东西都得有个兴趣，我们要放弃这样的想法。我们要带着任务来驱动学习，没有任务我们自己来制造任务，这样才能贯彻高效的学习。带着去教别人的心态参加学习，效果是最好的。

  • 唐僧在劝人的时候他是不觉得累的。^_^

  • 生活中的任务驱动，工作中的职业强迫，以教为学这些都是任务。

Summary

  • 成年人不需要培养兴趣，目标明确就开干。
  • 以教为学驱动方式应该是最好的。因为你得对你的学生负责，建立更好的个人品牌；这就促使你必须保证输出内容的质量。

怎么搞定拖延症

拖延症的表现和本质

  • 第一，任何人都有拖延症，只不过有些事他拖延，有些事他不拖延。因为一个人总不可能一直什么都不干嘛。
  • 第二，就算是自认为或者被别人认为有拖延症的人也没有被拖死，大家都还好好的活着呢。
  • 拖延的原因：如果任务就算拖延了对你也影响不大，那拖延概率就高了。

治疗办法

  • 将任务“严峻化”
  • 设定小目标，形成惯性

Summary

  • 首先，知道了任何人都有拖延症，这没什么，不会束缚你前进。
  • 课程中提到了任务严峻化、小目标；对于我来说将任务设置优先级即可，重点在怎么将复杂任务拆分成可执行的小任务（目标），并坚持把它完成好。
  • 关于小目标，我想说只有适合自己的才是最好的，哪怕它很小，关键是自己能坚持下来。

记得罗振宇老师分享过治疗拖延症的方法：

  • 把创造性工作分解成机械性劳动去完成。
  • 让种子飞一会。（增加思考时间）
  • 群体压力会消解拖延。让更多人监督自己。

训练专注力

平常的生活本身不能训练人的专注力，但是在平常的生活中去营造仪式感，这样会提升我们的注意力。

学习工具，你要买“贵”的。

你要相信，便宜没好货。图便宜浪费的是自己的时间。

这里贵相对的，对自己来说价格有点心疼就够了。 买了好的学习工具，不好好学不就是浪费了吗？哈哈，这招对穷人真有用。

  • 使用射灯，聚焦，专注。
  • 保证睡眠质量。
  • 运动健身也不可少。
  • 平时工作，可以听一些自己喜欢的歌，屏蔽嘈杂的环境来提高效率；
  • 困了就休息会儿，别硬撑。
  • 推荐一款我常用的保持专注的App：潮汐，App Store上搜索就有。

直奔大师，不必从基础开始

而对于成年人来讲，要开始学一样东西已经不容易了，一定要有一个明确的目标，并且找准这个学习的切入点。

  • 课程中的“直奔大师”，我的理解是：对自己要有信心，你可以搞定的。
  • 同志，时间不多了，快起来改bug。

给自己制造反馈

要想制造反馈，你必须先有成果，就是要基于已有的成果来发现问题，所以才要制造反馈。 自己给自己制造反馈的方法，我们可能要写一点东西出来，录制一点音频出来，甚至是录像。总之，是一定要有输出，或者说叫应用来作为前提的。

课程中提到三种反馈方法

  • 第一种写摘要，就是写summary。写summary不要脱稿写，要照着文本写；要用自己的语言来写。（所以才有了这篇学习笔记）
  • 第二种制造反馈的方法，给自己录音。
  • 第三种，给自己录像。

我是一个程序员，怎么给自己制造反馈呢？

  • 写文章并分享。 这个过程不仅能检查自己对知识点得掌握情况，还得把遇到问题都彻底搞明白了，毕竟是要分享的，不能误人子弟，也要对自己负责。
  • 解决工作中遇到的问题，不断优化并检验效果。也就是前文说的，高优先级，任务驱动。
  • 开发一个App。

怎样突破学习瓶颈

瓶颈是瓶颈，它不是极限。

  • 遇到瓶颈大多是两种情况：方法不对、心态问题。
  • 心态崩了咋整：得搞点啥建立信心才对啊。对程序员来说，写bug别犹豫，放开干！
  • 方法不对怎么办：Google、请教名师、大神、看书。
  • 课程中有提到: 遇到瓶颈可能是你的涉猎范围不够广，想要突破它你就要不断的积累。 我还是比较赞同的，正如查理芒格说的那句经典： 在手里拿着铁锤的人看来，世界就像一颗钉子。
  • 程序员也需要扩展自己的思维模型。推荐《穷查理宝典》。

精彩语句

  • 有些事真的是一通百通，你把一个东西拿下了，再去拿别的，会相对比较容易。
  • 凡走过必留下痕迹。学点什么都会有用的。

小结
